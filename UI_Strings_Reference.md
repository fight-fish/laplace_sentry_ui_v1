📄 Sentry UI v1.8 字串與位置總表 (UI String Map)
用途：開發 v2.0 時的「文案對照表」，確保搬家後訊息不流失、語氣不走樣。

【A. 專案列表區 (Project List)】

元件位置,顯示文字 (Text),備註
表頭 (Header),"UUID, 專案名稱, 監控狀態, 模式",順序與文字需一致
狀態欄 (Status),"監控中, 已停止",用於轉換代碼
模式欄 (Mode),"靜默, 互動",用於轉換代碼

【B. 右鍵選單 (Context Menu)】

動作,選單文字
手動更新,🔄 立即手動更新 (Manual Update)
修改,📝 修改專案...
刪除,🗑️ 刪除此專案...

【C. 詳情面板 (Detail Panel)】

情境,顯示內容
未選取時,尚未選取任何專案。
選取時,專案名稱：{name}監控狀態：{status}模式：{mode}專案路徑：{path}主寫入檔：{file}提示：雙擊左側列表可【啟動／停止】監控。

【D. 底部面板 (Bottom Panel)】

元件,文字
忽略說明,忽略設定區（暫時版）：- 哨兵忽略樣式：{patterns}- 目錄樹深度限制：{limit}
狀態列 (預設),狀態訊息：目前沒有任何訊息。
按鈕 1,編輯哨兵忽略清單… (目前禁用)
按鈕 2,編輯目錄樹忽略規則…

【E. 彈出視窗 (Dialogs & Messages)】

視窗類型,標題,內容 / 訊息
刪除確認,確認刪除,您確定要刪除專案「{name}」嗎？這將會：1. 停止該專案的哨兵 (若在運行)2. 從設定檔移除專案3. 清除相關日誌與暫存檔(不會刪除您的原始檔案)
手動更新 (進度),(狀態列),正在更新專案 '{name}'，請稍候...
手動更新 (成功),更新成功,專案 '{name}' 的目錄結構已更新至 Markdown。
手動更新 (失敗),更新失敗,(顯示錯誤訊息)
狀態切換 (成功),(狀態列),切換監控狀態成功：{name} 現在為 {status}。
狀態切換 (失敗),(狀態列),切換監控狀態失敗：找不到指定專案。

【F. 系統托盤 (System Tray)】

情境,文字
Tooltip (無監控),Sentry: 目前無監控
Tooltip (有監控),Sentry: {x} 個監控中 / {y} 個靜默中
右鍵選單,開啟控制台退出

【G. 視圖 A：哨兵之眼 (Sentry Eye)】
